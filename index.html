Parallel Processing Approach

  • Instead of recursively sorting and merging the entire array step-by-step like sequential merge sort, this algorithm splits the array into multiple smaller sections that can be sorted independently (in "parallel").
  • Different "threads" (simulated by different colors in this animation) work on different sections of the array simultaneously.
  • As threads finish sorting their sections, they are merged together progressively until the entire array is sorted.

Algorithm Steps:

  • Divide: Recursively split the array into smaller subarrays.
  • Assign Threads: Each split section is assigned a thread (color-coded).
  • Sort Independently: Each thread sorts its assigned section separately.
  • Merge Sections: After sorting, sections are merged back together in correct order.
  • Unify: After sorting, sections are merged back together in correct order.

Visualization Guide:

  • Each color represents a different thread working on a different section of the array.
  • During splitting, bars grouped under the same thread share a color.
  • As merging happens, sorted sections combine and colors reset to unify the array.
  • The animation completes when all bars are sorted and return to the primary theme colors (red/yellow).
